Integrando dispositivos y software

Integración de dispositivos y software Estamos tratando de proporcionar un par de métodos que brinden a los desarrolladores la capacidad de conectar sus dispositivos al tablero. Estos métodos son consistentes:

  • API apta para información bidireccional, como dispositivos de temperatura
  • API apta REST para información bidireccional, como api UDP del dispositivo de temperatura para información bidireccional, como dispositivos de temperatura
  • MQTT api, para comunicaciones multi direccionales, como enviar información y recibir órdenes
Nota: UDP y MQTT todavía están en el área experimental, por lo que no están disponibles en producción y su api y acceso de dominio pueden cambiar en el futuro cercano.

Obtener un token

Puedes encontrar tu Token de accesodentro de tu sección de configuraciones. Asegúrate de usar elToken de acceso solo para actualizarnos la información de tu dispositivo, como los cambios en la temperatura o la humedad, los usos de los dispositivos, etc. Puede haber otros tokens provistos y cada uno tiene su propio uso. Desde que tiene su token, puede hacer publicaciones y obtener solicitudes para el servidor de API incluido x-token en headers.

CloudDevice vs DataSource

Tenemos dos definiciones en el tablero para datos y dispositivos que son realmente simples de entender y sencillos.

CloudDevice es un dispositivo que puede crear en su tablero de administración, en sus Dispositivos > Crear un dispositivo. Este dispositivo representa un dispositivo físico o una parte de eso, y el propósito principal es ser utilizado para interacciones humanas, dibujar gráficos y hacer informes. Cada dispositivo, tiene una fuente de datos. Este origen de datos, es un dato en bruto, como un número para la temperatura que envía a api desde tu dispositivo. Todos los dispositivos deben estar conectados a una fuente de datos para ordenar la información.

Cuando envía sus datos con una identificación de fuente de datos, los recopilamos, pero si no hay ningún dispositivo para esta API, lo mostraremos en páginas de dispositivos, como dispositivo desconectado. No estamos sujetos a mantener esta información por mucho tiempo, si no hay un dispositivo asignado. Por lo tanto, es necesario crear un dispositivo basado en el dispositivo lo más rápido posible.

Puede conectar una fuente de datos a muchos dispositivos solo para dispositivos bidireccionales. No sabemos por qué podría usar esto, pero está disponible en caso de cualquier uso.
